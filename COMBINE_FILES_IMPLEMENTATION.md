# Combine Files Feature Implementation

## Overview
Added the ability to combine 2 or more result files into one, with automatic duplicate transaction ID prevention.

## Changes Made

### 1. Backend - Storage Service
**File:** `server/src/services/storageService.ts`

Added new method `mergeResults()`:
- Validates that at least 2 files are provided
- Loads all specified result files
- Merges accounts by account number (latest account data overwrites previous)
- Merges transactions with duplicate ID prevention using a `Set<string>` to track seen transaction IDs
- Skips any transaction with a duplicate ID that was already seen
- Creates a new merged result and saves it with timestamp
- Returns the filename of the merged result

**Key Features:**
- Duplicate Prevention: Uses a `Set` to track transaction IDs and only adds unique transactions
- Merge Strategy: Accounts are deduplicated by account number, transactions by ID
- Logging: Includes logs of merge operation with transaction tracking

### 2. Backend - API Route
**File:** `server/src/routes/scrapeRoutes.ts`

Added new endpoint `POST /results/merge`:
- Accepts `filenames` array (minimum 2 files) and `outputName` string
- Validates input parameters
- Calls `storageService.mergeResults()`
- Returns merged filename on success

### 3. Frontend - Hook
**File:** `client/src/hooks/useScraper.ts`

Added new hook `useMergeResults()`:
- Uses React Query mutation for server communication
- Sends POST request to `/results/merge` with filenames and output name
- Invalidates `scrapeResults` cache on success to refresh file list

### 4. Frontend - Component
**File:** `client/src/components/ResultsExplorer.tsx`

#### State Management:
- Added `showMergeModal` state to control modal visibility
- Added `mergeOutputName` state to store output filename input
- Added `isMerging` loading state from mutation hook

#### UI Elements:
- **Combine Button**: Appears in toolbar when 2+ files are selected
  - Only shows when `selectedFiles.length >= 2`
  - Shows loading spinner during merge operation
  - Uses emerald green styling to differentiate from other actions

- **Merge Modal**: 
  - Displays list of files to be combined
  - Input field for output filename
  - Confirmation message about duplicate removal
  - Cancel and Combine buttons
  - Supports Enter key to submit, Escape key to cancel

#### Handler Function:
- `handleMergeSubmit()`: Validates inputs and triggers merge operation
  - Logs merge operation details
  - Shows success/error messages via logger
  - Clears selection after successful merge
  - Auto-refreshes file list

## User Workflow

1. **Select Files**: User clicks on 2+ files in the file list (checkboxes select files)
2. **Combine Button Appears**: "Combine Files" button appears in toolbar
3. **Click Combine**: User clicks the button to open merge modal
4. **Enter Name**: User enters desired output filename (extension auto-added)
5. **Confirm**: User clicks "Combine" button
6. **Merge Process**: 
   - Files are loaded from server
   - Transactions are deduplicated by ID
   - Accounts are merged by account number
   - Result is saved with new filename
7. **File List Refreshed**: New combined file appears in file list
8. **Selection Cleared**: Selected files are deselected

## Duplicate Prevention Logic

The implementation prevents duplicate transactions using a **Set-based tracking system**:

```typescript
const seenTransactionIds = new Set<string>();
const mergedTransactions: any[] = [];

for (const txn of result.transactions) {
    if (!seenTransactionIds.has(txn.id)) {
        seenTransactionIds.add(txn.id);
        mergedTransactions.push(txn);
    }
}
```

This ensures:
- Each transaction ID appears only once in the merged result
- First occurrence of a transaction ID is kept
- Subsequent duplicates are silently skipped
- Time complexity is O(n) where n = total transactions

## API Contract

### Request
```json
POST /results/merge
{
  "filenames": ["file1.json", "file2.json", "file3.json"],
  "outputName": "combined_results"
}
```

### Response
```json
{
  "success": true,
  "filename": "merged_20260210_123456.json"
}
```

### Error Response
```json
{
  "success": false,
  "error": "Must provide at least 2 filenames to merge"
}
```

## Security Considerations

- File paths validated using `path.resolve()` to prevent directory traversal
- Only files within `RESULTS_DIR` are processed
- Output name generated by system to prevent path injection
- User input for output name is sanitized (spaces/special chars converted to underscores)
